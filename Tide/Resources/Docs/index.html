01

Overview

Tide Bot API is the contract between your backend and the Tide client. It supports message delivery, callback buttons, webhook push, long polling, and a local bot management screen in the app.

Important. Never ship a bot token inside the public iOS app. Store it in a server-side secret manager, or in Keychain only for local development.

02

Quick start

Start by calling /getMe. If the token is valid, the bot profile is returned and you can begin processing updates.

Terminal
curl -X POST \
  https://api.example.com/bot$TIDE_BOT_TOKEN/getMe \
  -H "Content-Type: application/json"
{
  "ok": true,
  "result": {
    "id": 1042001,
    "is_bot": true,
    "first_name": "Weather",
    "username": "weather_bot"
  }
}

04

Updates

An update is a normalized event object. Handle only the fields you need, then store the latest update ID so your consumer stays idempotent. 

{
  "id": 928001,
  "message": {
    "id": 41,
    "date": 1781791200,
    "chat": { "id": 712004, "kind": "private" },
    "from": { "id": 712004, "is_bot": false, "first_name": "Maya" },
    "text": "/start"
  }
}

message

Incoming text, media, command, or file.

edited_message

Content changed after the original send.

callback_query

User tapped an inline button.

channel_post

Bot published a post into a channel.

05

Webhooks

Webhooks are the preferred production delivery mode. Tide sends a JSON update plus the X-Tide-Bot-Secret header for verification.

Swift / Vapor
app.post("tide", "webhook") { request async throws -> HTTPStatus in
  guard request.headers.first(name: "X-Tide-Bot-Secret") == environment.secret else {
    throw Abort(.unauthorized)
  }
  let update = try request.content.decode(BotUpdate.self)
  try await updateHandler.handle(update)
  return .ok
}
  • Use HTTPS with a valid certificate.
  • Acknowledge updates quickly and offload slow work to a queue.
  • Deduplicate by update ID.
  • Verify the secret header before decoding business logic.

06

Methods

POST/bot<token>/getMe

Returns the bot profile and validates the token.

POST/bot<token>/getUpdates

Fetches updates in polling mode.

POST/bot<token>/sendMessage

Sends text, optional HTML or MarkdownV2 formatting, and a reply markup.

POST/bot<token>/setWebhook

Registers the webhook URL and secret.

POST/bot<token>/deleteWebhook

Turns webhook delivery off and restores polling.

POST/bot<token>/answerCallbackQuery

Stops the loading indicator for an inline button action.

07

Inline keyboards

Buttons can open a URL or send a short callback payload back to your bot. Keep sensitive state on the server, not inside callback_data.

08

Security

Least privilege

Give each bot only the permissions it truly needs.

Rotation

Rotate tokens immediately after any exposure.

Validation

Validate payload size, content type, and command shape.

Audit

Log update IDs and method names, never raw secrets.

10

Swift SDK

The app bundles a Swift actor-based client that uses async/await and Codable models for local bot workflows and preview data.

Swift
let client = TideBotAPIClient(
  token: environment.botToken,
  baseURL: URL(string: "https://api.example.com/")!
)

let keyboard = BotInlineKeyboardMarkup(inlineKeyboard: [[
  BotInlineKeyboardButton(text: "Open Tide", url: URL(string: "https://tide.app"))
]])

let message = try await client.sendMessage(
  chatID: update.message!.chat.id,
  text: "<b>Ready.</b> Everything works.",
  keyboard: keyboard
)
